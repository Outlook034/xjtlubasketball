# 扰动生成代码性能优化对比

## 原模板代码的性能问题

### ❌ 主要瓶颈

1. **单进程数据加载**
   ```python
   num_workers=0  # 无法并行加载图像
   ```
   - **影响**：CPU利用率低，I/O等待时间长
   - **速度损失**：2-4倍

2. **小批次大小**
   ```python
   batch_size=1  # 每次只处理一张图像
   ```
   - **影响**：无法充分利用批量处理优势
   - **速度损失**：10-20%

3. **参数解析错误**
   ```python
   parser.add_argument('--severity_levels', type=list, ...)  # ❌ 错误！
   ```
   - **问题**：`type=list` 无法正确解析命令行参数
   - **正确写法**：`type=int, nargs='+'`

4. **缺少进度显示**
   - 无法了解处理进度和剩余时间
   - 难以估算完成时间

5. **可能的重复目录创建**
   - 如果 `create_*` 函数内部每次都创建目录，会浪费大量时间

## ✅ 优化后的改进

### 1. 多进程数据加载
```python
parser.add_argument('--num_workers', type=int, default=4)
# 使用4个worker进程并行加载图像
```
- **速度提升**：2-4倍（取决于CPU核心数）
- **适用场景**：图像数量多、I/O密集型任务

### 2. 可配置批次大小
```python
parser.add_argument('--batch_size', type=int, default=1)
```
- 虽然当前 `create_*` 函数可能只支持 batch_size=1
- 但为未来优化预留了接口

### 3. 修复参数解析
```python
parser.add_argument('--severity_levels', type=int, nargs='+', default=[1, 2, 3, 4, 5])
```
- 正确解析多个整数参数
- 自动验证范围（1-5）

### 4. 进度条显示
```python
try:
    from tqdm import tqdm
    HAS_TQDM = True
except ImportError:
    HAS_TQDM = False
```
- 可选使用 tqdm 显示进度
- 如果没有安装，自动回退到普通迭代

### 5. 预先创建目录
```python
os.makedirs(args.save_path, exist_ok=True)
# 在开始处理前创建主目录
```
- 减少文件系统操作
- 如果 `create_*` 函数内部也创建目录，建议进一步优化

### 6. 错误处理
```python
try:
    # 尝试使用多进程
except Exception as e:
    # 自动回退到单进程
```
- 自动处理多进程失败的情况
- 确保代码在各种环境下都能运行

```bash

```

### 优化后
```bash
# 使用默认4个worker进程
python optimized_script.py --image_dir ./images --if_brightness --severity_levels 1 2 3 4 5

# 使用8个worker进程（如果CPU核心数足够）
python optimized_script.py --image_dir ./images --if_brightness --num_workers 8 --severity_levels 1 2 3 4 5

```

## 进一步优化建议

如果 `utils.py` 中的 `create_*` 函数支持批量处理，可以：

1. **增加批次大小**
   ```python

   ```

2. **预先创建所有目录**

   ```python
   from concurrent.futures import ThreadPoolExecutor
   # 并行处理不同severity级别
   ```

4. **缓存图像加载**
   ```python
   # 如果同一图像需要多种扰动，可以缓存加载的图像
   ```

## 注意事项

1. **多进程限制**：
   - Windows 需要使用 `spawn` 方法
   - Linux 可以使用 `fork` 方法
   - PIL Image 在多进程中有序列化问题，但 tensor 可以正常使用

2. **内存使用**：
   - 多进程会增加内存使用
   - 如果内存不足，减少 `num_workers`

3. **I/O 瓶颈**：
   - 如果存储设备很慢，多进程的优势会降低
   - 考虑使用SSD或更快的存储
