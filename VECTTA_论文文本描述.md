# VecTTA 完整流程文本描述（适合论文）

## 一、整体流程概述

VecTTA（Viewpoint Equivariance Consistency Test-Time Adaptation）的完整流程包含两个主要阶段：Clean伪标签生成阶段和测试时适应阶段。整个流程从KITTI测试数据集输入开始，经过模型适应过程，最终输出适应后的深度预测结果。

---

## 二、阶段1: Clean伪标签生成（预处理阶段）

### 2.1 输入数据
- **KITTI测试数据集**: 包含测试图像集合 {I_i}_{i=1}^N 和对应的相机内参 {K_i}_{i=1}^N
- 其中 I_i 表示第i个测试图像，K_i 表示对应的相机内参矩阵

### 2.2 模型推理
使用预训练的深度估计模型（未经过测试时适应）对测试集进行推理：
- 模型设置为评估模式（eval mode），不计算梯度
- 对每个测试图像 I_i，模型输出深度预测 depth_clean_i
- 所有预测结果收集为深度图数组

### 2.3 保存Clean伪标签
- 将预测结果保存为NumPy数组文件：clean_pred_disps.npy
- 该文件包含所有测试样本的深度预测，形状为 [N, H, W]
- Clean伪标签将作为后续测试时适应的参考监督信号

---

## 三、阶段2: VecTTA测试时适应

### 3.1 初始化阶段

对每个测试样本 (I_i, K_i)，首先进行VecTTA的初始化：

**3.1.1 加载预训练模型**
- 加载预训练的深度估计模型，包括编码器（Encoder）和解码器（Decoder）
- 模型参数初始化为预训练权重

**3.1.2 配置可更新参数**
根据 `update_mode` 参数选择需要更新的模型参数：
- **bn_only模式**: 只解冻所有BatchNorm层的参数，冻结其他所有参数
- **bn_decoder模式**: 解冻整个解码器的参数以及所有BatchNorm层
- **last_layers模式**: 只解冻最后几层卷积层（名称包含'disp'）和所有BN层
- **all模式**: 解冻所有模型参数

对于BatchNorm层，设置 `track_running_stats = False`，清空 `running_mean` 和 `running_var`，使其在测试时使用当前batch的统计信息。

**3.1.3 初始化优化器**
- 创建AdamW优化器，只优化可更新参数
- 学习率默认设置为 1e-4
- 优化器将用于后续的参数更新

### 3.2 适应循环

对每个测试样本执行T步（默认T=5）的迭代适应过程：

**3.2.1 模型预测深度**
- 将模型切换到训练模式（train mode），启用梯度计算
- 输入测试图像 I_i，模型输出深度预测 depth_t
- 深度预测的形状为 [1, 1, H, W]
- 保留计算图用于后续的反向传播

**3.2.2 采样虚拟位姿**
- 调用 `sample_virtual_poses()` 方法生成4个虚拟的微小相机位姿变换
- 每个位姿包含平移向量和旋转向量（轴角表示）
- 4个位姿模式分别为：
  - 位姿1: 沿X轴平移+0.1米，绕X轴旋转+0.05弧度
  - 位姿2: 沿X轴平移-0.1米，绕X轴旋转-0.05弧度
  - 位姿3: 沿Z轴平移+0.1米，绕X轴旋转+0.05弧度
  - 位姿4: 沿Y轴平移+0.1米，绕Z轴旋转+0.05弧度
- 这些微小的位姿变换用于构建几何一致性约束

**3.2.3 计算几何一致性损失**

对每个虚拟位姿 p_j (j = 1, 2, 3, 4)，执行以下步骤：

**步骤3.2.3.1: 深度回投影**
- 调用 `back_project_depth()` 方法
- 输入：当前深度预测 depth_t、相机内参 K_i、虚拟位姿 p_j
- 过程：
  1. 提取相机内参的3x3矩阵 K_3x3
  2. 初始化BackprojectDepth和Project3D模块
  3. 计算逆内参 inv_K = K_3x3^(-1)
  4. 从位姿构建变换矩阵 T（使用invert=True计算逆变换）
  5. 将深度图反投影到3D点云：cam_points = backproject(depth_t, inv_K)
  6. 将3D点通过变换矩阵T变换并投影到新视角：pix_coords = project(cam_points, K, T)
  7. 提取3D点的Z坐标（深度值）
  8. 使用双线性插值将深度值采样到新视角的像素位置：transformed_depth = grid_sample(z, pix_coords)
  9. 创建有效掩码，过滤投影在图像范围外的像素
- 输出：回投影后的深度图 back_depth_j，形状为 [1, 1, H, W]

**步骤3.2.3.2: 创建可见性掩码**
- 调用 `create_visibility_mask()` 方法
- 检查 depth_t 和 back_depth_j 的数值有效性（有限值且大于1e-6）
- 去除边界2像素区域（border_margin=2），避免边界畸变
- 输出：有效像素掩码 mask_j，形状为 [1, 1, H, W]

**步骤3.2.3.3: 计算尺度不变深度误差**
- 调用 `compute_scale_invariant_depth_error()` 方法
- 对深度值进行安全处理：clamp到最小值1e-6，避免log(0)
- 计算对数差：log_diff = log(depth_safe) - log(back_depth_safe)
- 计算方差形式的尺度不变损失：L_si = Var(log_diff) = E[log²(diff)] - E²[log(diff)]
- 该损失对全局尺度变化不敏感，只关注相对深度关系

**步骤3.2.3.4: 计算梯度一致性损失**
- 调用 `compute_gradient_consistency_loss()` 方法
- 计算 depth_t 和 back_depth_j 在X和Y方向的梯度：
  - gradient_x: 水平方向梯度（右像素 - 左像素）
  - gradient_y: 垂直方向梯度（下像素 - 上像素）
- 计算梯度差异：diff_x = |grad_x(depth_t) - grad_x(back_depth_j)|
- 计算梯度差异：diff_y = |grad_y(depth_t) - grad_y(back_depth_j)|
- 归一化得到平均梯度差异：L_grad = (diff_x.sum() + diff_y.sum()) / 有效梯度数量
- 该损失保持深度图的局部几何结构（边缘、表面）

**步骤3.2.3.5: 累积几何损失**
- 对当前位姿的损失：L_geo_j = L_si + 0.5 × L_grad
- 权重0.5用于平衡尺度不变损失和梯度一致性损失

**步骤3.2.3.6: 平均所有位姿的损失**
- 对所有4个虚拟位姿的几何损失求平均：
  L_geo_total = (1/4) × Σ_{j=1}^4 L_geo_j

**3.2.4 计算Clean伪标签损失（可选）**

如果提供了Clean伪标签 clean_depth_i：

**步骤3.2.4.1: 维度调整**
- 如果 clean_depth_i 是3维 [B, H, W]，扩展为4维 [B, 1, H, W]

**步骤3.2.4.2: Batch维度匹配**
- 如果 clean_depth_i 和 depth_t 的batch size不一致，取较小值并截断

**步骤3.2.4.3: 空间维度匹配**
- 如果 clean_depth_i 和 depth_t 的空间分辨率不同，使用双线性插值将 clean_depth_i 调整到与 depth_t 相同的分辨率

**步骤3.2.4.4: 计算L1损失**
- clean_loss = L1_loss(depth_t, clean_depth_i)
- 使用平均绝对误差作为损失函数

**步骤3.2.4.5: 加入总损失**
- 如果未提供Clean伪标签，clean_loss = 0

**3.2.5 计算总损失**
- 总损失公式：L_total = L_geo_total + 0.5 × L_clean
- Clean伪标签损失的权重为0.5，与几何损失平衡

**3.2.6 反向传播与优化**

**步骤3.2.6.1: 清零梯度**
- optimizer.zero_grad()，清除上一次迭代的梯度

**步骤3.2.6.2: 反向传播**
- L_total.backward()，计算损失对可更新参数的梯度

**步骤3.2.6.3: 梯度裁剪**
- 如果启用了梯度裁剪（grad_clip > 0），使用 clip_grad_norm_() 将梯度范数裁剪到阈值（默认1.0）
- 防止梯度爆炸，提高训练稳定性

**步骤3.2.6.4: 更新参数**
- optimizer.step()，根据梯度更新可更新参数
- 使用AdamW优化器的更新规则

**3.2.7 早停检查（可选）**

如果启用了早停机制（early_stop=True）：
- 记录当前损失值 loss_val
- 如果 loss_val < best_loss（损失下降）：
  - 更新 best_loss = loss_val
  - 重置 patience_counter = 0
- 否则（损失未下降）：
  - patience_counter += 1
- 如果 patience_counter >= early_stop_patience（默认3）：
  - 提前终止适应循环，跳出迭代

**3.2.8 循环继续**
- 重复步骤3.2.1到3.2.7，直到完成所有T步迭代或触发早停

### 3.3 最终预测

**3.3.1 切换模型模式**
- 将模型切换回评估模式（eval mode）
- 禁用梯度计算，使用torch.no_grad()上下文

**3.3.2 最终深度预测**
- 使用适应后的模型参数对测试图像 I_i 进行最终预测
- 输出：depth_final_i，形状为 [1, 1, H, W]
- 这是经过测试时适应优化后的深度预测结果

---

## 四、输出结果

### 4.1 单个样本输出
- 对每个测试样本 I_i，输出适应后的深度预测 depth_final_i

### 4.2 完整测试集输出
- 所有测试样本的适应后深度预测集合 {depth_final_i}_{i=1}^N
- 可用于后续的深度估计评估指标计算（如绝对相对误差、均方根误差等）

---

## 五、关键设计要点

### 5.1 几何一致性约束
- 通过虚拟位姿变换构建几何闭环约束，利用深度预测应该满足的几何一致性
- 无需真实深度标签，仅利用几何先验知识

### 5.2 多损失组合
- **尺度不变损失**: 对全局尺度变化不敏感，适合单目深度估计的尺度模糊性
- **梯度一致性损失**: 保持局部几何结构，提供细粒度约束
- **Clean伪标签损失**: 提供稳定参考，防止适应过程偏离太远

### 5.3 选择性参数更新
- 只更新部分参数（如BN层、Decoder），减少计算量，提高效率
- 避免过拟合，保持预训练模型的知识

### 5.4 早停机制
- 防止过拟合，当损失不再下降时提前停止
- 提高适应效率

## 六、损失函数总结

### 6.1 总损失公式

```
L_total = L_geo_total + 0.5 × L_clean
```

其中：

```
L_geo_total = (1/4) × Σ_{j=1}^4 [L_si(d, d_back^j) + 0.5 × L_grad(d, d_back^j)]
```

### 6.2 各项损失说明

- **L_si**: 尺度不变深度误差，使用对数差的方差形式
- **L_grad**: 梯度一致性损失，保持局部几何结构
- **L_clean**: Clean伪标签的L1损失，权重0.5

### 6.3 损失权重
- 尺度不变损失权重：1.0
- 梯度一致性损失权重：0.5
- Clean伪标签损失权重：0.5

---

## 七、完整流程总结

VecTTA的完整流程从KITTI测试数据集输入开始，首先使用预训练模型生成Clean伪标签并保存。然后对每个测试样本，初始化VecTTA适应器，配置可更新参数和优化器。在适应循环中，模型预测深度，采样虚拟位姿，通过深度回投影计算几何一致性损失，可选地计算Clean伪标签损失，然后反向传播优化参数。经过T步迭代（或早停）后，在评估模式下进行最终预测，输出适应后的深度结果。整个过程无需重新训练模型，仅通过测试时的快速适应即可提升深度估计的准确性。
